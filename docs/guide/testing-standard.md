# Testing Standard

## 目的

解析微分が正しいことを数値的に保証する。

## 最低要件

- 勾配: 中央差分または複素ステップ
- 許容誤差: `atol` と `rtol` の併用
- テスト点: 典型点 + 境界近傍（制約境界そのものは回避）
- 再現性: 乱数 seed 固定

## 推奨要件

- HVP検証: `(grad(x + eps * v) - grad(x)) / eps` と `hvp(x, v)` の一致
- 複数の `eps` 値で安定性確認
- 不安定領域での安定化手法（`eps` ガード等）を明記

## チェック項目

- ドメイン違反入力がテストに混入していない
- 誤差が閾値内である
- 境界近傍ケースで発散しない
